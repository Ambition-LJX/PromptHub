# PromptHub 部署指南

> 本文档为 PromptHub (AI 提示词管理系统) 提供多种部署方案推荐，帮助你快速将项目上线访问。

---

## 方案对比一览

| 方案 | 难度 | 成本 | MySQL | 推荐指数 | 最适合场景 |
|------|------|------|-------|----------|-----------|
| **Railway** | 简单 | 付费（$5/月起） | 内置 | ⭐⭐⭐⭐⭐ | 个人/团队使用，最省心 |
| **Vercel + PlanetScale** | 简单 | 免费/付费 | 兼容MySQL | ⭐⭐⭐⭐ | 习惯 Vercel 的用户 |
| **Coolify + VPS** | 中等 | 云服务器费用 | 自建 | ⭐⭐⭐ | 完全自主可控 |
| **Docker + VPS** | 中等 | 云服务器费用 | 自建 | ⭐⭐ | 有服务器运维经验 |

**综合推荐：Railway** —— 开箱即用，一条命令几乎不需要配置，Next.js + MySQL 一站式搞定。

## 推荐方案一：Railway（最省心）

Railway 是目前最方便的 Next.js + MySQL 部署平台，平台自动处理构建、部署、环境变量和数据库。

### 前置准备

1. 注册 [Railway](https://railway.app)（推荐用 GitHub 账号登录）
2. 安装 Railway CLI：

```bash
npm install -g @railway/cli
```

3. 登录：

```bash
railway login
```

### 步骤 1：创建 MySQL 数据库

1. 在 Railway 控制台点击 **New Project** → **Empty Project**
2. 点击 **Add a Database** → 选择 **MySQL**
3. 等待 MySQL 创建完成，点击数据库 → **Variables**，复制 `MYSQLHOST`、`MYSQLPORT`、`MYSQLPASSWORD`、`MYSQLUSER`
4. 在 Railway 的数据库详情页，点击 **Connect** → 查看连接信息

### 步骤 2：修改数据库连接配置

Railway 提供的 MySQL 连接需要使用 `mysql2` 驱动的一些特殊配置。创建 `.env.production` 文件（**不要提交到 Git**）：

```env
DATABASE_URL="mysql://<USER>:<PASSWORD>@<HOST>:<PORT>/railway"
JWT_SECRET="<随机生成的强密钥，建议64位以上>"
JWT_EXPIRES_IN="7d"
```

> 生成随机密钥的方法：
> ```bash
> node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
> ```

### 步骤 3：初始化数据库表结构

Railway 的 MySQL 不支持 `prisma migrate`，需要手动导入 SQL。先在本地生成 SQL：

```bash
# 在项目根目录（PromptHub文件夹）
npx prisma migrate dev --name init
```

然后在 Railway 控制台：
1. 点击你的 MySQL 数据库
2. 点击 **Query** 标签
3. 打开 `prisma/prompt_hub_mysql.sql` 文件，复制全部内容粘贴到 Railway 的查询窗口执行

### 步骤 4：部署 Next.js 应用

在 Railway 控制台：
1. 进入你的 Project
2. 点击 **New** → **Next.js**（或直接从 GitHub 部署）
3. 连接你的 GitHub 仓库 `prompt-hub`
4. 配置环境变量（从步骤 2 的 `.env.production` 中复制）
5. Railway 自动检测 Next.js 构建命令（`npm run build`）

或者用 CLI 部署：

```bash
cd PromptHub
railway init
railway add --database   # 选择之前创建的 MySQL
railway up
```

### 步骤 5：配置域名（可选）

Railway 免费版提供一个随机子域名（如 `odd-water-1234.railway.app`）。

如需自定义域名：
1. Railway 控制台 → 你的服务 → **Settings** → **Networking**
2. 添加自定义域名，配置 DNS 解析

### 注意事项

- Railway 免费额度每月 $5（500小时），足够个人使用
- MySQL 数据库在免费版有 1GB 存储限制
- Railway 的 MySQL 不支持外键约束（已处理）
- 如果遇到连接问题，确保 `ssl` 参数正确，Railway MySQL 连接通常需要 SSL

## 推荐方案二：Vercel + PlanetScale（免费方案）

Vercel 是 Next.js 官方推荐的托管平台，配合 PlanetScale（免费 MySQL 兼容数据库）可实现完全免费部署。

### 前置准备

1. 注册 [Vercel](https://vercel.com)（用 GitHub 登录）
2. 注册 [PlanetScale](https://planetscale.com)（免费开源项目可用免费套餐）
3. 将项目推送到 GitHub 仓库

### 步骤 1：创建 PlanetScale 数据库

1. PlanetScale 控制台 → **New database** → 输入名称 `prompt-hub`
2. 选择免费套餐 **Hobby**
3. 创建完成后，进入数据库 → **Branches** → 选择 `main` 分支
4. 点击 **Connect** → 选择 **Prisma**，复制 `DATABASE_URL`

### 步骤 2：配置 Prisma 连接

PlanetScale 使用的是 MySQL 兼容协议，但不支持外键。修改 Prisma Schema 中的连接字符串：

```env
DATABASE_URL="mysql://<USER>:<PASSWORD>@<HOST>/prompt_hub?sslaccept=strict"
```

同时，由于 PlanetScale 不支持 `migrate`（使用分支模式），需要在本地生成 SQL 后通过 Console 手动执行：

```bash
# 本地生成迁移 SQL
npx prisma migrate dev --name init --create-only
```

然后在 PlanetScale Console 中执行生成的 SQL。

### 步骤 3：推送到 Vercel

```bash
# 安装 Vercel CLI
npm i -g vercel

cd PromptHub
vercel
```

按提示操作：
- Set up and deploy? `Y`
- Which scope? 选择你的账号
- Link to existing project? `N`（如果是新项目）
- Project name? `prompt-hub`
- Directory? `./`
- Override settings? `N`

### 步骤 4：配置环境变量

Vercel 控制台 → 你的项目 → **Settings** → **Environment Variables**，添加：

```
DATABASE_URL = <PlanetScale的连接字符串>
JWT_SECRET = <随机生成的强密钥>
JWT_EXPIRES_IN = 7d
```

然后重新部署：

```bash
vercel --prod
```

### 步骤 5：配置自定义域名

Vercel 控制台 → **Domains** → 添加你的域名，Vercel 会自动配置 SSL 证书。

## 方案三：Coolify + VPS（完全自主可控）

Coolify 是一个开源的自动化部署平台（类似 Heroku 的开源替代），可以一键部署 Next.js + MySQL 到自己的 VPS 服务器。

### 适用场景

- 已有云服务器（阿里云、腾讯云、AWS 等）
- 不想受平台限制，完全自主控制
- 长期运行，数据留存在自己服务器

### 前置准备

1. 一台有公网 IP 的 VPS（建议 2核4G 以上）
2. 域名（如没有可先用 IP 访问）
3. 服务器上安装 Coolify

### 步骤 1：安装 Coolify

```bash
# 在服务器上执行（需要 Docker）
curl -fsSL https://get.coolify.io | bash
```

安装完成后，访问 `http://<服务器IP>:3000` 完成初始化配置。

### 步骤 2：在 Coolify 中创建项目

1. Coolify 控制台 → **Projects** → **Create Project**
2. 添加一个 **Application**
3. 连接 GitHub 仓库
4. 设置构建命令：
   - Build Command: `npm ci && npx prisma migrate deploy && npm run build`
   - Run Command: `npm start`
5. 设置环境变量（DATABASE_URL、JWT_SECRET、JWT_EXPIRES_IN）

### 步骤 3：配置 MySQL

Coolify 控制台 → **Databases** → **Add Database** → 选择 **MySQL**：
- 数据库名：`prompt_hub`
- 用户名：`prompt_hub`
- 设置密码

创建后获取连接信息，填入应用的环境变量：

```env
DATABASE_URL="mysql://prompt_hub:<密码>@<Coolify-MySQL地址>:3306/prompt_hub"
```

### 步骤 4：部署

点击 **Deploy**，Coolify 会自动：
1. 克隆代码
2. 安装依赖
3. 运行数据库迁移
4. 构建 Next.js
5. 启动服务

### 步骤 5：配置 Nginx 反向代理和 SSL

Coolify 自带反向代理，但你也可以手动配置 Nginx：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:<应用端口>;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_cache_bypass $http_upgrade;
    }
}
```

使用 Let's Encrypt 获取免费 SSL：

```bash
certbot --nginx -d your-domain.com
```

### VPS 推荐配置

| 配置项 | 推荐 |
|--------|------|
| CPU | 2核以上 |
| 内存 | 4GB 以上 |
| 系统 | Ubuntu 22.04 LTS |
| 磁盘 | 40GB SSD 以上 |
| 带宽 | 5Mbps 以上 |

## 方案四：Docker + VPS（传统方式）

如果你是 Docker 重度用户，可以用 Dockerfile 部署。

### 在项目根目录创建 `Dockerfile`：

```dockerfile
# 使用 Node.js 20 Alpine 镜像
FROM node:20-alpine AS base

# 安装依赖阶段
FROM base AS deps
WORKDIR /app
COPY package.json package-lock.json* ./
RUN npm ci

# Prisma 生成阶段
FROM base AS prisma
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY prisma ./prisma
RUN npx prisma generate

# 构建阶段
FROM base AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY --from=prisma /app/node_modules/.prisma ./node_modules/.prisma
COPY . .
RUN npm run build

# 运行阶段
FROM base AS runner
WORKDIR /app
ENV NODE_ENV production

RUN addgroup --system --gid 1001 nodejs
RUN adduser --system --uid 1001 nextjs

COPY --from=builder /app/public ./public
COPY --from=builder --chown=nextjs:nodejs /app/.next/standalone ./
COPY --from=builder --chown=nextjs:nodejs /app/.next/static ./.next/static

USER nextjs
EXPOSE 3000
ENV PORT 3000

CMD ["node", "server.js"]
```

### 在 `next.config.ts` 中添加 standalone 输出：

```ts
const nextConfig: NextConfig = {
  allowedDevOrigins: ["192.168.31.46"],
  output: "standalone",
};
```

### 创建 `.dockerignore`：

```
node_modules
.next
.git
.env
.env.local
Dockerfile
docker-compose.yml
```

### 创建 `docker-compose.yml`：

```yaml
version: '3.8'

services:
  app:
    build: .
    restart: always
    ports:
      - "3000:3000"
    environment:
      DATABASE_URL: ${DATABASE_URL}
      JWT_SECRET: ${JWT_SECRET}
      JWT_EXPIRES_IN: "7d"
    depends_on:
      db:
        condition: service_healthy

  db:
    image: mysql:8.0
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: root_password
      MYSQL_DATABASE: prompt_hub
      MYSQL_USER: prompt_hub
      MYSQL_PASSWORD: db_password
    volumes:
      - mysql_data:/var/lib/mysql
      - ./prisma/prompt_hub_mysql.sql:/docker-entrypoint-initdb.d/init.sql
    ports:
      - "3306:3306"
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5

volumes:
  mysql_data:
```

### 部署到 VPS：

```bash
# 克隆项目到服务器
git clone https://github.com/your-username/prompt-hub.git
cd prompt-hub/PromptHub

# 设置环境变量
cp .env.example .env
nano .env  # 修改 DATABASE_URL 和 JWT_SECRET

# 构建并启动
docker-compose up -d --build

# 查看日志
docker-compose logs -f
```

## 部署后必做清单

### 安全检查

- [ ] `.env` 文件**不要**提交到 Git（`.gitignore` 中确认已排除）
- [ ] 修改默认的 `JWT_SECRET`（使用 `openssl rand -hex 64` 生成强密钥）
- [ ] MySQL 密码不要使用弱密码
- [ ] 启用 HTTPS（所有部署平台默认自带 SSL）
- [ ] 如果使用 VPS，确保防火墙只开放必要端口（80, 443, 22）

### 域名解析

- [ ] 将域名 A 记录指向服务器 IP（或 CNAME 指向平台提供的域名）
- [ ] 配置 SSL 证书（Let's Encrypt 或平台自动）
- [ ] 设置 www 到主域名的 301 重定向（可选）

### 数据库

- [ ] 确认数据库迁移已成功执行（所有表已创建）
- [ ] 如有初始数据，执行 `npm run db:seed` 导入
- [ ] 开启数据库定期备份（Railway/PlanetScale 自动备份，自建需手动）

### 监控

- [ ] 设置平台告警（内存、CPU、磁盘使用率）
- [ ] 配置错误日志监控（Vercel 自带，Railway 可集成 Sentry）
- [ ] 记录访问日志

## 常见问题排查

### 1. 页面空白或白屏

```bash
# 清除浏览器缓存，或检查控制台错误
# 可能是环境变量未正确设置
```

### 2. 数据库连接失败

```
Error: Can't connect to MySQL server
```

- 检查 `DATABASE_URL` 是否正确
- 确认数据库服务已启动
- 检查防火墙是否放行了 MySQL 端口（自建时）
- MySQL 8.0 可能需要 `sslaccept=strict` 参数

### 3. 401/403 认证错误

- 确认 `JWT_SECRET` 在构建和运行时一致（修改后需重新部署）
- 清除浏览器 localStorage 中的旧 token

### 4. Prisma 迁移问题

```
Error: P1001: Can't find a `mysql` database
```

- 确认数据库已创建
- 确认用户有创建表的权限

### 5. 内存不足（VPS 部署）

```bash
# 添加 swap
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

### 6. 静态资源 404

Next.js 部署后静态资源路径问题，检查 `next.config.ts` 的 `basePath` 和 `assetPrefix` 配置。

## 生产环境建议

### 性能优化

- 开启 Redis 缓存（大型团队使用）
- 配置 CDN 加速静态资源
- 启用 Next.js Image Optimization

### 数据备份

| 部署方案 | 备份方式 |
|----------|----------|
| Railway | 自动每日备份，可在设置中下载 |
| PlanetScale | 自动备份，可恢复任意时间点 |
| 自建 MySQL | 使用 `mysqldump` 定时备份到对象存储 |

### 备份命令（自建 MySQL）：

```bash
mysqldump -u root -p prompt_hub > backup_$(date +%Y%m%d).sql

# 上传到对象存储（阿里云 OSS 为例）
ossutil cp backup_20240501.sql oss://your-bucket/backup/
```
