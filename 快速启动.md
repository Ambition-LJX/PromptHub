# 快速启动

本指南帮助你快速搭建并运行 PromptHub 项目。

---

## 一、环境要求

- Node.js >= 18.17
- MySQL 8.0（项目默认使用 MySQL 作为数据库）
- npm 或 pnpm

---

## 二、项目依赖安装

### 1. 进入项目目录

```bash
cd PromptHub
```

### 2. 安装依赖（使用国内镜像源）

```bash
npm install --registry=https://registry.npmmirror.com
```

如果使用 pnpm：

```bash
pnpm install --registry=https://registry.npmmirror.com
```

---

## 三、环境变量配置

### 1. 复制环境变量模板文件

```bash
cp .env.example .env
```

### 2. 修改 .env 文件中的数据库配置

```bash
# 使用文本编辑器打开 .env 文件，修改以下配置
DATABASE_URL="mysql://用户名:密码@localhost:3306/prompt_hub"
JWT_SECRET="your-super-secret-jwt-key-change-in-production"
JWT_EXPIRES_IN="7d"
```

将 `用户名` 和 `密码` 替换为你的 MySQL 连接信息。

---

## 四、数据库初始化

### 方式一：使用 Prisma 迁移（推荐）

#### 1. 登录 MySQL 并创建数据库

```bash
mysql -u root -p
```

在 MySQL 命令行中执行：

```sql
CREATE DATABASE prompt_hub CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

#### 2. 生成 Prisma Client

```bash
npm run db:generate
```

#### 3. 运行数据库迁移

```bash
npm run db:migrate
```

#### 4. 填充测试数据

```bash
npm run db:seed
```

### 方式二：直接导入 SQL 文件

如果你希望跳过 Prisma 迁移步骤，直接使用已有的 SQL 初始化脚本初始化数据库，可以按以下步骤操作：

#### 1. 导入 SQL 文件

```bash
mysql -u root -p < prisma/prompt_hub_mysql.sql
```

这条命令会执行以下操作：

- 自动创建 `prompt_hub` 数据库（如果不存在）
- 创建所有数据表
- 插入默认管理员账号（[admin@prompthub.local](mailto:admin@prompthub.local) / password123）
- 插入 10 条提示词模板示例数据
- 插入 3 个项目流程模板和 8 个阶段
- 插入 1 个工作集示例

#### 2. 生成 Prisma Client（可选）

如果你后续需要通过 Prisma 操作数据库，仍需运行：

```bash
npm run db:generate
```

> **注意**：方式一（Prisma 迁移）和方式二（SQL 导入）只需选择一种。方式一会自动调用 Prisma Migrate 创建表结构，方式二使用手工 SQL 脚本，两者的最终效果相同。

## 八、注意事项

1. 如果 npm install 安装缓慢，建议始终使用 `--registry=https://registry.npmmirror.com` 参数指定国内镜像源。
2. 如果 MySQL 连接失败，请确认：
  - MySQL 服务已启动
  - `.env` 文件中的 `DATABASE_URL` 配置正确
  - MySQL 用户拥有创建表和读写数据的权限
3. 重置数据库的方法：
  ```bash
   # 删除迁移文件和数据库，重新初始化
   npx prisma migrate reset --registry=https://registry.npmmirror.com
  ```
